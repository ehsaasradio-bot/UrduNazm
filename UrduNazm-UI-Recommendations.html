UrduNazm UI/UX Recommendations

Analysis of Rekhta.org + 2026 Design Trends → Actionable Design Direction

Part 1

Rekhta.org — What They Do Well & Where We Improve

Rekhta is the largest Urdu poetry platform with 30,000+ ghazals/nazms from 2,500+ poets across 3 centuries. They have massive content, built-in glossary, multi-script support (Urdu, Devanagari, Roman), and a mobile app. Here's the honest assessment:

Rekhta Weaknesses (Our Opportunity)

  • Cluttered, information-dense homepage — feels overwhelming
  • Dated visual design — not premium or modern
  • Too many navigation items and content types competing
  • Urdu typography could be more beautiful — feels functional, not artistic
  • No AI-powered features (search, chat, recommendations)
  • Dark mode not available
  • Slow page loads due to heavy content
  • Mobile experience feels like a squeezed desktop site
  • No personalization or smart curation
  • Ads and promotions dilute the reading experience

Rekhta Strengths (Learn From)

  • Comprehensive poet profiles with era and bio
  • Multi-script toggle (Urdu, Hindi, Roman) — we do Urdu + English
  • Built-in word meanings on click — glossary feature
  • Audio recitations of poetry
  • Explorer section for browsing by mood/topic/occasion
  • Strong SEO — ranks #1 for most Urdu poetry queries
  • Popular choice curation (trending poems)
  • Deep content categorization (ghazal, nazm, rubai, qita, etc.)
  • Poet filtering by era and location
  • E-book library integration

Our positioning: UrduNazm is NOT trying to be Rekhta. We are a curated, premium, modern poetry experience — think Apple meets poetry. Less content, more beauty. Quality over quantity. AI-powered. Dark-first. Fast.

Part 2

2026 Design Trends We're Adopting

Based on research from Figma, UX Collective, and leading design publications — filtered for what makes sense for a poetry platform.

1. Dark-First Design

Dark mode is the default in 2026. Poetry reads beautifully on dark backgrounds — gold on black is timeless. Light mode available as toggle. Dynamic theming based on system preference.

2. Kinetic Typography

Hero text that builds letter by letter. Urdu verses that fade in line by line. Words that gently animate on scroll. Text IS our content — make it the star.

3. Scroll-Driven Storytelling

Long homepage that unfolds as you scroll. Subtle parallax. Progressive reveals. Each section enters with purpose. No jarring page loads — smooth transitions.

4. Micro-Interactions

Bookmark heart animations. Button ripples. Hover states on cards. Loading shimmer effects. Every tap gives feedback. Interfaces feel alive, not static.

5. AI Integration

AI chat for poetry exploration. Smart search that understands context. "Find me a ghazal about longing" — conversational discovery. AI-powered translation assistance.

6. Generous Whitespace

Poetry needs room to breathe. Large margins. Ample line spacing. No visual clutter. Every element earns its place. Apple-level spatial hierarchy.

7. Variable Fonts

Single font files with continuous weight ranges. Smooth weight transitions on hover. Urdu Nastaliq rendered at optimal size with proper line height (2.0-2.4x).

8. Glassmorphism + Depth

Frosted glass navbar. Subtle layered depth on cards. Background blur on modals. Creates visual hierarchy without hard borders. Modern, premium feel.

Part 4

Typography System

Usage Font Weight Notes
English headings Inter 600-700 Tight tracking (-0.02em). Clean, modern.
English body Inter 400 Line height 1.6-1.7. Comfortable reading.
Urdu headings Noto Nastaliq Urdu 700 RTL. Line height 2.2-2.4. Large sizing (1.3x English).
Urdu poetry body Noto Nastaliq Urdu 400 RTL. Line height 2.4+. Generous spacing between verses.
Code / meta System mono 400 Dates, tags, small labels.

Key RTL rules: Always set dir="rtl" on Urdu containers. Use CSS logical properties (margin-inline-start, not margin-left). Urdu font size should be ~1.3x English equivalent for visual parity. Avoid bold for long Urdu text — connected letterforms become harder to read.

Part 5

Page-by-Page Design Direction

Navbar

Sticky, frosted-glass effect (backdrop-blur). Logo left, nav links center, search + theme toggle + login right. On mobile: hamburger → full-screen drawer with large touch targets. Transparent on hero, solid on scroll.

Homepage

  1. Hero: Full-viewport. Large Urdu verse center-aligned with subtle fade-in animation. English translation below in smaller text. Minimal CTA: "Explore Poetry". Dark gradient background with faint decorative pattern.
  2. Featured Poems: Horizontal scroll or 3-column grid. Card shows Urdu excerpt (2-3 lines), poet name, category badge. Hover lifts card with shadow. Gold accent on hover border.
  3. Featured Poets: Circular avatar images with name below (Urdu primary, English secondary). Horizontal scroll on mobile. Subtle parallax on desktop.
  4. Verse of the Day: Full-width section with large Urdu typography on elegant background. Changes daily. Shareable.
  5. Recent Blog: 2-3 cards. Cover image, title, date, reading time. Clean layout.
  6. CTA / AI Chat Promo: "Ask about any poem" — teaser for the AI chat feature. Floating chat button persistent across site.

Poets Page

Grid of poet cards (image, name Urdu + English, era). Search bar at top. Filter by era (Classical, Modern, Contemporary). Cards have hover zoom on image. Clean, gallery-like feel. Each card links to poet profile.

Poet Profile

Large hero with poet image + name + dates. Bio section with tabs (Urdu | English). Below: grid of their poems. Bookmark poet feature. Share button. Related poets sidebar.

Poems Listing

Left sidebar filters (category, poet, era). Main area: poem cards in list or grid view (toggle). Each card: Urdu title, first 2 lines preview, poet name, category badge. Sort by: newest, popular, poet.

Poem Detail

The most important page. Full Urdu text centered with beautiful Nastaliq rendering. Line-by-line layout with generous spacing. Toggle for English translation (appears below each line or side-by-side). Bookmark button (heart with animation). Share. Poet attribution with link. Related poems below.

Blog

Clean magazine layout. Large featured post at top. Below: 2-column grid of posts. Each card: cover image, title, excerpt, date, read time. Individual post: long-form with good typography, images, pull quotes.

Search

Cmd+K modal (2026 standard). Full-screen on mobile. Search across poems, poets, blog. Results grouped by type with icons. Recent searches. Debounced live results. AI-enhanced: "ghazals about the night" works.

AI Chat

Floating button bottom-right. Opens slide-up panel or dedicated /chat page. Streaming responses. System prompt: Urdu poetry expert. Features: explain a verse, translate, find similar poems, discuss a poet's style. Clean chat bubbles with proper RTL for Urdu responses.

Auth & Profile

Google login only — single button, no form. Profile page: avatar, name, bookmarked poems grid. Minimal. No settings bloat.

Footer

4-column on desktop: About, Explore (Poets, Poems, Blog), Features (Search, AI Chat), Social links. Urdu tagline. Copyright. Subtle top border. Clean, not crowded.

Part 6

Performance & SEO Strategy

Server Components

All listing and detail pages are Server Components by default. Zero client JS for content pages. Only interactive elements (search, chat, bookmarks) use "use client".

Font Loading

Inter via next/font (self-hosted). Noto Nastaliq via Google Fonts with display=swap and preconnect. Font subsetting for Urdu to reduce payload.

Image Optimization

All images via next/image with automatic WebP/AVIF. Poet images: fixed sizes with blur placeholders. Blog covers: responsive srcSet.

Metadata

Dynamic metadata per page. Open Graph images auto-generated with poem excerpt. Twitter cards. hreflang tags for Urdu content. JSON-LD structured data for poems and poets.

Sitemap & Robots

Dynamic sitemap.ts that lists all poems, poets, blog posts. Proper robots.txt. Canonical URLs. No duplicate content between Urdu/English views.

Core Web Vitals

Target: LCP < 2.5s, FID < 100ms, CLS < 0.1. Skeleton loaders for async content. No layout shift from font loading. Streaming SSR for long lists.

Key Technical Decisions
