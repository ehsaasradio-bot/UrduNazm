03

Supabase Schema & Seed Data

Database Done

Design and create all database tables, relationships, indexes, RLS policies, and seed with initial data.

  • poets — id, name_en, name_ur, slug, bio_en, bio_ur, image_url, born, died, era
  • poems — id, title_en, title_ur, content_ur, content_en, poet_id, category_id, slug, featured, fts
  • categories — id, name_en, name_ur, slug
  • blogs — id, title_en, title_ur, content_en, content_ur, excerpt_en, slug, cover_image, category, author_name, published
  • bookmarks — id, user_id, poem_id, created_at
  • profiles — id (FK auth.users), display_name, avatar_url, role + auto-create trigger
  • RLS policies: public read for poems/poets/blogs, authenticated write for bookmarks
  • SQL migration file + full-text search index
  • Seed data: 5 categories, 6 poets, 12 poems, 1 blog
  • TypeScript types + server-side query helpers
