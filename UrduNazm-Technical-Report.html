Security Audit

Score: 74/100 — 1 critical vulnerability requires immediate attention

🚨
CRITICAL — XSS Vulnerability in Blog Content
File: src/app/blog/[slug]/page.tsx:95 — dangerouslySetInnerHTML={{ __html: blog.content_en }} renders raw HTML from the database without sanitization. If malicious HTML is inserted into blog content, it executes in the browser (stored XSS). Fix: install isomorphic-dompurify and sanitize before render.
✅ Passed
✅
Supabase RLS Policies
Row-level security on all tables. Users can only access their own bookmarks/profile.
✅
SSR Auth (no cookies exposed)
@supabase/ssr handles server-side session — no tokens in client JS.
✅
Middleware Route Protection
/bookmarks redirects unauthenticated users to /login with next param.
✅
Environment Variables
.env.local in .gitignore, keys never committed to GitHub.
✅
HTTPS Only
Vercel enforces HTTPS — all traffic encrypted in transit.
✅
No SQL Injection Risk
Supabase JS client uses parameterized queries — no raw SQL exposed.
⚠️ Issues Found
🔴
XSS — Blog HTML Content [CRITICAL]
dangerouslySetInnerHTML without DOMPurify. Fix immediately with isomorphic-dompurify.
🟠
No Rate Limiting on /api/chat
Chat API can be abused to run up OpenAI costs. Add Vercel rate limiting or upstash/ratelimit.
🟠
TypeScript Build Errors Ignored
ignoreBuildErrors: true masks type safety. Should fix underlying types and remove.
🟡
No Content Security Policy (CSP)
Missing CSP headers. Add via next.config.ts headers() to restrict script sources.
🟡
img tag in UserMenu
Using raw <img> for Google avatar instead of Next.js <Image>. Minor — no security risk.
